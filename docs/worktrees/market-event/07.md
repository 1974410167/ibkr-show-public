你现在在 `ibkr_show` 项目中工作。当前任务是实现“市场事件中台”的第七块：**前台重点事件页面**。

注意：本任务只做前端页面和前端 API 调用，不做 Agent 联动，不做新的后端 Provider，不修改交易/订单/账户逻辑。

## 二、强约束

1. 不要实现 Agent 联动。
2. 不要修改交易决策 Agent、每日复盘 Agent、持仓分析 Agent。
3. 不要修改 IBKR、交易、订单、账户逻辑。
4. 不要在前端保存或展示任何明文 API Key。
5. 不要调用后台 key 配置接口，除非是在后台配置页面已有功能中。
6. 前台只能使用第六块公开查询 API。
7. 不要提交 git commit。
8. 页面风格必须贴合现有前端 UI、路由、状态管理、请求封装、组件规范。
9. 所有接口失败要有空状态和错误状态，不要白屏。
10. 移动端/小屏幕至少保证可用，不要求极致适配。

## 十六、性能要求

1. 默认 limit=50。
2. 详情按需请求，不要一次性拉所有详情。
3. 风险总览、列表、sources 可以并发请求。
4. 总览页卡片失败时静默降级。
5. 筛选输入 keyword / symbol 需要 debounce。
6. 避免无限循环请求。
