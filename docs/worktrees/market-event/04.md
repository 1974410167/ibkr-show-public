你现在在 `ibkr_show` 项目中工作。当前任务是实现“市场事件中台”的第四块：**同步任务框架 + sync log**。

注意：本任务只做同步任务框架和同步日志，不接真实外部数据源，不实现前台页面，不实现 Agent 联动。Provider 仍然可以使用第三块里的空实现。

## 二、目标

实现一个统一的 Market Event Sync 框架：

```text
MarketEventSyncService
├── sync_calendar_events
├── sync_event_values
├── sync_news_events
├── sync_corporate_events
├── sync_market_holidays
├── sync_market_status
├── sync_trading_sessions
└── sync_all
```

每次同步必须写入：

```text
market_event_sync_runs
```

并支持：

```text
1. 手动触发同步
2. 指定 source_code 同步
3. 指定 capability 同步
4. 指定时间范围同步
5. 幂等 upsert
6. 同步日志统计
7. 失败记录
8. 安全重试基础
```

本阶段 Provider 可以返回空结果或 SKIPPED，但框架必须完整。

## 七、Sync Run 生命周期

同步开始时创建 `market_event_sync_runs`：

```text
status = RUNNING
started_at = now
source_code
provider_name
sync_type
range_start
range_end
metadata
```

同步结束时更新：

```text
status
finished_at
total_count
created_count
updated_count
skipped_count
failed_count
error_message
error_detail
metadata
```

异常要求：

```text
1. 单个 Provider 失败，不应导致所有 Provider 同步中断，除非是 sync_one 明确要求。
2. 失败必须记录 error_message。
3. error_detail 不得包含 API key、token、账户信息。
4. SKIPPED 也要记录 sync_run，方便后台展示为什么没跑。
```

## 八、幂等 Upsert 设计

实现统一 repository/service 方法：

```text
upsert_occurrence(input: MarketEventUpsertInput) -> UpsertResult
upsert_values(occurrence_id, values)
upsert_impacts(occurrence_id, impacts)
upsert_news_links(occurrence_id, news_links)
```

要求：

```text
1. occurrence 以 dedupe_key 唯一去重。
2. dedupe_key 不存在时，使用第一块的工具方法生成。
3. raw_payload_hash 变化时，更新 raw_payload 和 updated_at。
4. source_event_id 可以为空，但 dedupe_key 必须有。
5. values 按 occurrence_id + value_type + label + period 去重或更新。
6. impacts 按 occurrence_id + symbol + asset_class + market + sector + industry 去重或更新。
7. news_links 按 occurrence_id + source_code + news_id 或 url 去重。
8. dry_run=true 时不写数据库，只返回统计。
```

返回 UpsertResult：

```text
UpsertResult:
- created_count
- updated_count
- skipped_count
- failed_count
- occurrence_ids
- errors
```

## 十四、测试要求

```text

8. raw_payload_hash 变化时更新 occurrence。
9. values / impacts / news_links upsert 能去重。
10. 默认时间窗口是 timezone-aware UTC。
```
