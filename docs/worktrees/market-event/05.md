## 五、BLS Provider 接通

### 1. 数据范围

BLS 负责：

```text
CPI
PPI
NONFARM_PAYROLLS
UNEMPLOYMENT_RATE
JOLTS
```

### 2. 日历数据

优先接 BLS Release Calendar ICS：

```text
https://www.bls.gov/schedule/news_release/bls.ics
```

解析 ICS，转换成未来事件 occurrence。

重点映射：

```text
Consumer Price Index -> CPI
Producer Price Index -> PPI
Employment Situation -> NONFARM_PAYROLLS / UNEMPLOYMENT_RATE
Job Openings and Labor Turnover Survey -> JOLTS
```

要求：

```text
1. scheduled_at 存 UTC。
2. scheduled_timezone 记录 America/New_York。
3. source_code = BLS。
4. category = MACRO。
5. importance:
   - CPI: CRITICAL
   - Employment Situation: CRITICAL
   - PPI: HIGH
   - JOLTS: HIGH
6. source_url 使用 BLS release 页面或 ICS URL。
7. raw_payload 保存脱敏后的 ICS 事件字段。
```

### 3. 实际值 / 历史值

接 BLS API，使用后台配置的 BLS API Key。

先实现最小 series 映射：

```text
CPI:
- CUUR0000SA0

UNEMPLOYMENT_RATE:
- LNS14000000

NONFARM_PAYROLLS:
- CES0000000001

PPI:
- WPUFD4

JOLTS:
- JTSJOL
```

说明：

```text
1. series id 如项目中已有更准确映射，可调整。
2. BLS API 返回的是时间序列，不一定直接对应某个 occurrence。
3. 本次只需要把最新值转换成 MarketEventValueInput，并尽量匹配最近 occurrence。
4. 如果无法可靠匹配 occurrence，只记录 provider metadata，不要乱写错误值。
```

### 4. BLS health_check

```text
1. source disabled -> SKIPPED
2. key 未配置 -> SKIPPED
3. key 已配置 -> 请求一个极小 API 查询
4. 成功 -> SUCCESS
5. 失败 -> FAILED，并记录脱敏错误
```

## 七、Fed Provider 接通

Fed 不需要 API key。

### 1. 数据范围

```text
FOMC_RATE_DECISION
FOMC_MINUTES
FOMC_SEP
FED_SPEECH 可先不做或作为后续 TODO
```

### 2. FOMC Calendar

接 Fed 官方 FOMC calendar 页面：

```text
https://www.federalreserve.gov/monetarypolicy/fomccalendars.htm
```

解析会议日期。

要求：

```text
1. 支持未来会议。
2. 识别是否带 SEP。
3. FOMC_RATE_DECISION 默认时间如果页面没有明确时间，可设为会议最后一天 18:00 UTC 左右，并标记 is_confirmed_time=false。
4. category = FED。
5. importance = CRITICAL。
6. scheduled_timezone = America/New_York。
7. source_code = FED。
8. raw_payload 保存页面解析后的公开字段，不保存整页大 HTML。
```

### 3. Minutes

如果页面能解析 Minutes 日期，则生成 FOMC_MINUTES occurrence。

如果无法稳定解析，本次只做 FOMC_RATE_DECISION + FOMC_SEP，Minutes 留 TODO。

### 4. health_check

请求 FOMC calendar 页面，成功返回 SUCCESS。

## 八、Longbridge Provider 接通

优先使用项目已有 Longbridge 接入方式。

如果项目已通过 MCP / SDK / OpenAPI / 内部 service 接入 Longbridge，请复用，不要重新造一套。

### 1. 数据范围

Longbridge 负责：

```text
财报：EARNINGS
分红：DIVIDEND
拆股：SPLIT
IPO：IPO
休市日：MARKET_CLOSED / HALF_TRADING_DAY
交易时段：TRADING_SESSION_CHANGE
新闻：NEWS / POLICY / TARIFF / EXPORT_CONTROL / BTC_EVENT / MSTR_BTC_EVENT
```

### 2. 财报 / 分红 / 拆股 / IPO

调用 Longbridge calendar 能力，等价于：

```text
finance_calendar(category=report)
finance_calendar(category=dividend)
finance_calendar(category=split)
finance_calendar(category=ipo)
```

要求：

```text
1. 支持按 market 和时间范围查询。
2. 对返回结果做本地时间范围过滤，因为 Longbridge 可能返回窗口外事件。
3. 统一转换为 MarketEventUpsertInput。
4. 财报事件要解析：
   - symbol
   - company name
   - period
   - market_time: before / after / unknown
   - estimate_eps
   - estimate_revenue
5. 分红事件要解析：
   - ex_date
   - record_date
   - payment_date
   - dividend amount
6. 拆股事件要解析 split ratio。
7. IPO 事件要解析 symbol、market、listing date。
```

### 3. 市场休市日

调用 Longbridge closed calendar / trading_days。

要求：

```text
1. market 支持 US / HK / CN。
2. 全日休市 -> MARKET_CLOSED。
3. 半日交易 -> HALF_TRADING_DAY。
4. importance = MEDIUM 或 HIGH。
5. market 字段必须写入。
```

### 4. 交易时段

调用 trading_session。

转换为 TRADING_SESSION_CHANGE 或保存为 raw market session event。

如果不适合落 occurrence，可以放 metadata 并留给 API 层展示。

### 5. 新闻事件候选

调用：

```text
news(symbol)
```

持仓 symbol 从项目现有持仓数据服务读取；如果读取持仓会侵入太大，本次先支持传入 symbols 参数。
Longbridge Provider 不实现 keyword news search；主题/关键词新闻后续由 GenericNewsProvider 或其他专用新闻源实现。

新闻转换要求：

```text
1. 新闻不要全部变成高影响事件。
2. 默认 category=NEWS。
3. importance=LOW / MEDIUM。
4. 新闻 url + title + published_at 作为 dedupe 输入。
5. 同时写 market_event_news_links。
```

### 6. health_check

检查 Longbridge 配置或调用一个轻量公开接口。

不得调用交易、下单、账户修改接口。

## 十、错误处理

每个 Provider 要区分：

```text
1. source disabled -> SKIPPED
2. credential missing -> SKIPPED
3. network timeout -> FAILED
4. parse error -> PARTIAL_SUCCESS 或 FAILED
5. empty result -> SUCCESS，total_count=0
```

不能把“没有事件”当成失败。
