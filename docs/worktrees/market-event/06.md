你现在在 `ibkr_show` 项目中工作。当前任务是实现“市场事件中台”的第六块：**事件查询 API**。

注意：本任务只做后端查询 API，不做前台重点事件页面，不做 Agent 联动。前端页面会在第七块实现。

## 六、API 设计

### 1. GET /api/market-events

返回事件列表。

返回结构建议：

```json id="uu6hzg"
{
  "items": [],
  "total": 0,
  "limit": 50,
  "offset": 0
}
```

每个 item 使用 `MarketEventListItem`：

```text id="iucwoz"

source_code
country

has_forecast_value

impacts

```

默认不返回：

```text id="lkijv4"
raw_payload
error_detail
encrypted_value
API key
```

### 2. GET /api/market-events/today

返回今天事件。

参数：

```text id="i577qx"
display_timezone，默认 Asia/Shanghai 或项目现有用户时区
market
symbol
importance
```

逻辑：

```text id="m5euku"
1. 按 display_timezone 计算当天 00:00:00 到 23:59:59。
2. 转换成 UTC 查库。
3. 默认只返回 MEDIUM / HIGH / CRITICAL，也可由参数控制。
```

### 3. GET /api/market-events/upcoming

返回未来事件。

参数：

```text id="vh7jnn"
days，默认 7
importance
symbol
market
category
```

默认：

```text id="yxw4xk"
未来 7 天，高影响优先。
```

### 4. GET /api/market-events/risk-summary

返回风险概览，给总览页和重点事件首页使用。

返回结构建议：

```json id="kfe8kq"
{
  "today": {
    "risk_level": "HIGH",
    "critical_count": 1,
    "high_count": 2,
    "medium_count": 4
  },
  "next_7_days": {
    "risk_level": "CRITICAL",
    "critical_count": 2,
    "high_count": 5,
    "medium_count": 8
  },
  "next_30_days": {
    "risk_level": "HIGH",
    "critical_count": 5,
    "high_count": 12,
    "medium_count": 20
  },
  "top_events": []
}
```

风险等级规则：

```text id="befiqw"
如果窗口内 critical_count >= 1 -> CRITICAL
否则 high_count >= 2 -> HIGH
否则 medium_count >= 3 -> MEDIUM
否则 LOW
```

允许后续调整，但本次先做明确规则。

### 5. GET /api/market-events/calendar

### 7. GET /api/market-events/symbol/{symbol}

返回影响某个标的的事件。

参数：



## 八、时区要求

必须使用 timezone-aware datetime。

要求：

```text id="o4msf9"
1. start_at / end_at 如果带 timezone，转换 UTC 查询。
2. 如果不带 timezone，按 display_timezone 或默认 Asia/Shanghai 解释。
3. 返回 scheduled_at 为 UTC ISO。
4. 返回 display_time 为 display_timezone 下的时间字符串或对象。
5. calendar 按 display_timezone 分组。
```

## 十、风险概览规则

实现一个独立函数：

```text id="m4svag"
calculate_event_risk_level(events) -> LOW / MEDIUM / HIGH / CRITICAL
```

规则：

```text id="nn7qep"
CRITICAL:
- 至少一个 CRITICAL 事件

HIGH:
- 至少两个 HIGH 事件
- 或一个 HIGH 且三个以上 MEDIUM

MEDIUM:
- 至少三个 MEDIUM
- 或一个 HIGH

LOW:
- 其他情况
```

同时输出：

```text id="55s1it"
critical_count
high_count
medium_count
low_count
top_events
```
