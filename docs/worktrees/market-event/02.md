## 六、后端 API 设计

新增后台 API，路径请遵循项目现有 admin API 风格。建议：

```text
GET    /api/admin/market-event-sources
GET    /api/admin/market-event-sources/{source_code}
PUT    /api/admin/market-event-sources/{source_code}
PUT    /api/admin/market-event-sources/{source_code}/credential
DELETE /api/admin/market-event-sources/{source_code}/credential
POST   /api/admin/market-event-sources/{source_code}/test
```

### 1. GET list

返回所有事件数据源配置。

返回字段包含：

```text
source_code
source_name
description
enabled
priority
apply_url
doc_url
requires_api_key
credential_key_name
credential_configured
masked_value
last_check_at
last_check_status
last_error
updated_at
```

禁止返回：

```text
encrypted_value
明文 key
```

### 2. GET detail

返回单个 source 配置详情。规则同上，不返回明文 key。

### 3. PUT source config

允许更新：

```text
enabled
priority
description
apply_url
doc_url
```

不允许更新：

```text
source_code
encrypted_value
```

### 4. PUT credential

请求体示例：

```json
{
  "credential_key": "api_key",
  "value": "用户输入的真实 key"
}
```

要求：

```text
1. value 必须后端加密保存。
2. 返回时只返回 masked_value。
3. 不要记录明文 value 到日志。
4. 如果 value 为空，返回参数错误。
```

对不同数据源默认 credential_key：

```text
BLS: api_key
BEA: api_key
FRED: api_key
```

虽然 BEA 官方叫 UserID，项目内部可以统一 credential_key 为 api_key，展示名叫 BEA API Key / UserID。

### 5. DELETE credential

删除或禁用对应 source credential。

要求：

```text
1. 不删除 source 本身。
2. 删除后 credential_configured=false。
3. 操作可重复执行。
```

### 6. POST test

只做轻量测试，不做完整同步。

本阶段可以实现为“配置级测试”，不要拉取大量数据。

推荐测试策略：

```text
BLS:
- 如果已配置 key，校验 key 格式非空。
- 可以尝试请求 BLS API 的一个极小测试接口；如果项目当前不允许联网测试，则先返回 SKIPPED，并说明将在 Provider 阶段实现真实测试。

BEA:
- 如果已配置 key，校验 key 格式非空。
- 可以尝试一个最小 metadata 请求；如果不实现真实网络请求，返回 SKIPPED。

FRED:
- 如果已配置 key，校验 key 格式非空。
- 可以尝试一个轻量 endpoint；如果不实现真实网络请求，返回 SKIPPED。

FED / ISM:
- 无 key，返回 SUCCESS，说明无需 API key。

LONGBRIDGE:
- 复用项目现有 Longbridge 配置检查；如果本阶段不接，返回 SKIPPED。
```

测试结果要写回 source：

```text
last_check_at
last_check_status
last_error
```

状态建议：

```text
SUCCESS
FAILED
SKIPPED
```


```text

4. 代码注释说明生产环境必须配置 secret。
```

可支持配置名：

```text
APP_SECRET_KEY
CONFIG_ENCRYPTION_KEY
```

按项目已有 settings 风格决定。

### 2. 脱敏

新增或复用 mask 工具：

```text
abcdef123456 -> ****3456
短 key -> ****
空值 -> null
```

要求：

```text
1. 前端只能看到 masked_value。
2. 日志只能看到 masked_value。
3. API response 不能包含明文。
```
