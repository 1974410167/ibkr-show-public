你现在在 `ibkr_show` 项目中工作。当前任务是实现“市场事件中台”的第一块：**数据库模型 + 枚举 + 统一事件 schema**。

注意：本任务只做底层数据模型和统一 schema，不做后台配置页面、不做 API、不接真实数据源、不做同步任务、不做前端页面、不做 Agent 联动。

## 一、背景

项目准备新增“重点事件 / 市场事件中台”能力，用于统一管理：

1. 宏观经济事件：CPI、PPI、非农就业、失业率、JOLTS、PCE、GDP、ISM PMI。
2. 美联储事件：FOMC、SEP 点阵图、会议纪要、Fed 官员讲话。
3. 公司事件：财报、分红、拆股、IPO、投资者日。
4. 市场事件：休市日、半日交易、交易时段变化。
5. 新闻事件：持仓相关新闻、宏观突发新闻、政策事件、关税、出口管制、BTC/MSTR 相关事件。

长期架构是多数据源：

```text
BLS        # CPI / PPI / 非农 / 失业率 / JOLTS
BEA        # GDP / PCE / 个人收入支出
FRED       # 宏观时间序列备用/校验
FED        # FOMC / SEP / Minutes / Fed speech
ISM        # PMI
LONGBRIDGE # 财报 / 分红 / 拆股 / IPO / 休市 / 新闻
MANUAL     # 后台手动补充事件
```

你要先搭好最终架构的数据模型，避免后续返工。

## 四、本次需要新增的核心概念

### 1. Event Source：事件数据源

表示一个数据源，例如 BLS、BEA、FRED、FED、ISM、LONGBRIDGE、MANUAL。

### 2. Event Definition：事件定义

表示一种长期存在的事件类型，例如：

```text
CPI
PPI
NONFARM_PAYROLLS
UNEMPLOYMENT_RATE
JOLTS
PCE
GDP
FOMC_RATE_DECISION
ORCL_EARNINGS
AAPL_DIVIDEND
US_MARKET_CLOSED
```

Definition 是“事件模板 / 事件类型”，不是某一次具体发布。

### 3. Event Occurrence：事件实例

表示某一次具体事件，例如：

```text
2026-06 CPI 发布
2026-06-10 ORCL 财报
2026-06-17 FOMC 利率决议
2026-11-27 US 半日交易
```

### 4. Event Value：事件数值

表示某一次事件的前值、预期值、实际值、修正值等，例如：

```text
previous
forecast
actual
revised
```

### 5. Event Impact：事件影响映射

表示事件可能影响哪些资产、行业、市场或持仓标的。

### 6. Event Analysis：事件分析

预留给后续 LLM 生成事前分析、事后复盘，但本次不调用 LLM。

### 7. Event Sync Run：同步记录

预留给后续同步任务记录每次拉取数据源的结果。

## 八、幂等去重设计

本次要实现一个纯函数或工具方法，用于生成 `dedupe_key`。

建议位置按项目现有工具目录决定，例如：

```text
backend/app/market_events/utils.py
```

逻辑建议：

```text
dedupe_key = hash(source_code + event_type + scheduled_at + title + market + period + symbols)
```

要求：

1. 输入相同，输出稳定。
2. 不包含 API key 或隐私数据。
3. 有基础单元测试。
4. 后续 Provider 可以复用。

同时实现 `raw_payload_hash` 工具方法：

```text
raw_payload_hash = stable_json_hash(raw_payload)
```

要求 JSON key 排序后 hash，避免同一 payload 因字段顺序变化导致 hash 不一致。
