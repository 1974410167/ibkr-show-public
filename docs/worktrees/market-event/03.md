你现在在 `ibkr_show` 项目中工作。当前任务是实现“市场事件中台”的第三块：**Provider 抽象 + BLS / BEA / FRED / Fed / ISM / Longbridge 空实现**。

注意：本任务只做 Provider 架构和空实现，不接真实外部接口，不做同步任务，不做事件查询 API，不做前台重点事件页面，不做 Agent 联动。

## 一、前置背景

前两块已经或应该完成：

```text
01_database_model_and_event_schema:
- 市场事件数据库模型
- 枚举
- 统一事件 schema
- dedupe_key / raw_payload_hash 工具

- 测试连接接口骨架
```

本次要建立最终架构的 Provider 层，后续所有数据源都通过统一接口输出标准化事件。

## 二、目标

新增统一 Provider 抽象：

```text
MarketEventProvider
├── BlsProvider
├── BeaProvider
├── FredProvider
├── FedProvider
├── IsmProvider
├── LongbridgeProvider
└── ManualProvider
```

每个 Provider 本阶段只做：

```text
1. 声明 provider_name / source_code / capabilities
2. 实现统一方法签名
3. 返回空结果或 SKIPPED 结果
4. 读取自身配置状态
5. 校验是否启用
6. 校验是否配置必要 key
7. 不调用真实外部接口
```

## 八、Provider Result Schema

新增统一结果结构。

建议：

```text
ProviderHealthCheckResult:
- source_code
- provider_name
- status: SUCCESS / FAILED / SKIPPED
- message
- checked_at
- metadata

ProviderFetchResult:
- source_code
- provider_name
- status: SUCCESS / PARTIAL_SUCCESS / FAILED / SKIPPED
- events: list[MarketEventUpsertInput]
- values: list[MarketEventValueInput]
- impacts: list[MarketEventImpactInput]
- news_links: list[MarketEventNewsLinkInput]
- total_count
- error_message
- metadata
```

注意：

1. 本阶段空实现返回 `SKIPPED` 或空 `SUCCESS`，不要假造真实事件。
2. Provider 返回的是统一 schema，不直接返回 ORM model。
3. 真实外部 raw payload 本阶段为空或简单 metadata。

## 九、配置读取能力

Provider 需要能够读取对应 source 配置状态。

例如：

```text
1. source 是否 enabled
2. 是否 requires_api_key
3. credential 是否 configured
4. masked_value 是否存在
```

但本阶段严禁读取或返回明文 key。

建议新增：

```text
MarketEventSourceConfigService
```

或复用第二块已有 service。

Provider 初始化或执行时应判断：

```text
如果 source disabled:
  返回 SKIPPED, message="source disabled"

如果 source requires_api_key 且 credential 未配置:
  返回 SKIPPED, message="credential not configured"

如果 source 无需 key:
  允许 health_check 返回 SUCCESS 或 SKIPPED
```
